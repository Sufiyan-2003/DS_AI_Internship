Understanding why tuples cannot be changed in Python

Concept Explanation

In Python, a tuple is an immutable data type. This means once a tuple is created, its elements cannot be modified, added, or removed.

This is useful when you want to ensure data integrity - like game settings that shouldn't change during gameplay.

Game Resolution Example

Imagine you're setting up the screen resolution for a game. This setting should not change during gameplay.

screen_res = (1920, 1080)

If you try to change it: screen_res[0] = 1280

You'll get a TypeError because tuples are immutable!

Key Points

  • Tuples are immutable - cannot be changed after creation
  • Lists are mutable - can be modified after creation
  • Use tuples for data that shouldn't change (constants, configurations)
  • Use lists for data that needs to be modified
  • Tuples can be used as dictionary keys (lists cannot)
